# 수료증 대량 생성 서비스 — 프로젝트 설계도

> 버전 0.1 | 2026-03-11 | 마풀랩

---

## 1. 프로젝트 개요

### 목적
수료증 템플릿 이미지 위에 엑셀 데이터(성명, 기간, 발급번호)를 매핑하여 한 번에 최대 100건의 수료증 이미지를 자동 생성하는 로컬 도구.

### 핵심 요구사항

| 항목 | 내용 |
|------|------|
| 변동 필드 | 성명, 기간(시작~종료), 발급번호 |
| 고정 필드 | 이수시간, 진행강사, 본문, 서명 등 (템플릿에 포함) |
| 템플릿 | 현재 1종, 향후 다중 템플릿 확장 예정 |
| 출력 형식 | PNG (1차), PDF (향후 확장) |
| 실행 환경 | 로컬 PC (Python 스크립트) |
| 사용자 | 1인 (마풀랩) |
| 최대 처리량 | 1회 100건 미만 |

### 4-1. 엑셀 입력 파일 (participants.xlsx)

|----|--------|------|------|
| A | 발급번호 | GW_202602_001 | 고유값, 파일명에 사용 |
| B | 성명 | 홍길동 | |
| C | 기간_시작 | 2026-03-14 | 날짜 형식 |
| D | 기간_종료 | 2026-03-14 | 날짜 형식 |

### 4-2. config.json 구조

```json
{
  "template": {
    "name": "default",
    "image_path": "templates/default.png",
    "output_dir": "output",
    "output_format": "png",
    "output_filename": "{발급번호}_{성명}"
  },
  "fonts": {
    "name": {
      "path": "fonts/NotoSansKR-Regular.otf",
      "size": 42,
      "color": [51, 51, 51]
    },
    "period": {
      "path": "fonts/NotoSansKR-Regular.otf",
      "size": 30,
      "color": [51, 51, 51]
    },
    "issue_number": {
      "path": "fonts/NotoSansKR-Regular.otf",
      "size": 20,
      "color": [102, 102, 102]
    }
  },
  "fields": {
    "발급번호": {
      "x": 230,
      "y": 68,
      "font_key": "issue_number",
      "format": "{value}"
    },
    "성명": {
      "x": 255,
      "y": 440,
      "font_key": "name",
      "format": "{value}"
    },
    "기간": {
      "x": 255,
      "y": 498,
      "font_key": "period",
```

## 5. 핵심 로직 설계

### 5-1. 실행 흐름 (generate.py)

```
1. config.json 로드
2. 엑셀 파일 읽기 (openpyxl)
3. 데이터 유효성 검사
   - 필수 필드 누락 확인
   - 발급번호 중복 확인
   - 날짜 형식 검증
4. 행(row)별 반복:
   a. 템플릿 이미지 복사 (PIL Image.copy())
   b. 각 필드에 대해:
      - config에서 좌표·폰트·포맷 조회
      - ImageDraw.text()로 텍스트 렌더링
   c. 출력 파일명 생성: {발급번호}_{성명}.png
   d. PNG 저장
5. 결과 요약 출력 (성공/실패 건수)
```

### 5-2. 날짜 포맷 변환

엑셀의 날짜(2026-03-14)를 수료증 표기 형식으로 변환:

```
2026-03-14 → "2026년 3월 14일"
```

- `datetime.strftime` 활용
- 월·일 앞의 0 제거 (03→3, 07→7)

### 5-3. 텍스트 렌더링 주의사항

- 한글 폰트 필수 (시스템 기본 폰트는 한글 미지원 가능)
- Noto Sans KR 또는 원본 수료증에 사용된 폰트 권장
- 텍스트 정렬: 좌표는 텍스트 시작점(좌상단) 기준
- 안티앨리어싱: Pillow 기본 적용

## 6. CLI 인터페이스

### 6-1. 기본 사용법

```bash
# 수료증 생성
python generate.py --data data/participants.xlsx

# 특정 템플릿 지정
python generate.py --data data/participants.xlsx --template workshop_b

# 출력 폴더 지정
python generate.py --data data/participants.xlsx --output ./2026년3월_수료증

# 특정 행만 생성 (테스트용)
python generate.py --data data/participants.xlsx --rows 1-3
```

### 6-2. 좌표 보정 모드 (Calibrate)

템플릿이 바뀌거나 처음 세팅할 때, 각 필드의 정확한 좌표를 잡기 위한 보조 기능:

```bash
python generate.py --calibrate
```

- 템플릿 위에 각 필드 좌표 위치를 빨간 십자(+) 마커로 표시한 `_calibrate.png`를 생성
- 마커 위치를 확인하고 config.json의 좌표를 조정하는 방식

## 7. 기술 스택

| 구분 | 선택 | 이유 |
|------|------|------|
| 언어 | Python 3.10+ | 이미지 처리·엑셀 라이브러리 풍부 |
| 이미지 처리 | Pillow (PIL) | 가볍고 텍스트 오버레이에 적합 |
| 엑셀 읽기 | openpyxl | .xlsx 네이티브 지원 |
| CLI | argparse (내장) | 외부 의존성 없음 |
| 향후 PDF | Pillow → PDF 변환 또는 reportlab | PNG 생성 후 변환 방식 |

### requirements.txt

```
Pillow>=10.0.0
openpyxl>=3.1.0
```

## 8. 에러 처리

| 상황 | 처리 방식 |
|------|-----------|
| 엑셀 파일 없음 | 에러 메시지 + 종료 |
| 필수 열 누락 | 누락된 열 이름 표시 + 종료 |
| 발급번호 중복 | 경고 표시 후 계속 진행 (덮어쓰기) |
| 성명 비어있음 | 해당 행 스킵 + 경고 |
| 폰트 파일 없음 | 에러 메시지 + 대체 폰트 안내 |
| 템플릿 이미지 없음 | 에러 메시지 + 종료 |
| 출력 폴더 없음 | 자동 생성 |

## 11. 선행 준비 사항

개발 시작 전 준비가 필요한 항목:

1. **템플릿 이미지**: 텍스트가 없는 빈 템플릿 PNG (성명·기간·발급번호 영역이 비어있는 버전)
   - 현재 이미지에는 "홍길동", 날짜 등 샘플 텍스트가 들어있으므로, Illustrator/Figma에서 해당 텍스트를 제거한 버전 필요
2. **폰트 파일**: 수료증에 사용된 폰트의 .otf 또는 .ttf 파일
   - 성명, 기간, 발급번호 각각 어떤 폰트를 사용했는지 확인
3. **샘플 엑셀**: 테스트용 수료자 데이터 3~5건
